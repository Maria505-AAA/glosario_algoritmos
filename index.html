Mi glosario personal de Pensamiento algoritmico

Por: Nicolas Orjuela Yepes y Maria Ximena Lopez Aullon

Glosario de términos

A continuación encontrarás las definiciones de las principales palabras y expresiones relacionadas con pensamiento algoritmico


Lógica

Capacidad de estructurar razonamientos de forma coherente y ordenada para resolver problemas. En el contexto de pensamiento algorítmico, la lógica es fundamental para identificar relaciones entre datos, formular condiciones, y garantizar que un conjunto de pasos produzca resultados esperados de manera consistente y sin ambigüedades en cualquier situación.

Secuencia

Conjunto de instrucciones ordenadas que se ejecutan de manera lineal dentro de un algoritmo. Cada paso depende del resultado del anterior y proporciona los datos necesarios para el siguiente. Es crucial para garantizar la continuidad del flujo lógico en la solución de problemas, evitando saltos inesperados que interrumpan el proceso.

Abstracción

Proceso de identificar los elementos esenciales de un problema eliminando detalles irrelevantes o secundarios. Ayuda a enfocarse en las características principales, reduciendo la complejidad y permitiendo diseñar soluciones más claras y generalizadas. En el pensamiento algorítmico, la abstracción es clave para modelar problemas grandes de manera comprensible.

Modularidad

Principio que organiza un problema o programa en unidades independientes llamadas módulos, cada una con una funcionalidad específica. Esto facilita la programación, el análisis y la depuración al permitir trabajar en partes más pequeñas y manejables. Además, mejora la reutilización de código, ya que los módulos pueden integrarse en diferentes contextos.

Recursión

Técnica donde una función o procedimiento se llama a sí mismo con un subconjunto del problema original, resolviendo iterativamente hasta llegar a un caso base que sea fácil de calcular. Es útil para problemas que pueden dividirse en subproblemas idénticos, como en algoritmos para recorrer estructuras de datos como árboles o grafos.

Iteración

Proceso de repetir un conjunto de instrucciones o pasos en un algoritmo hasta alcanzar un objetivo o condición definida. Cada repetición se denomina iteración, y es esencial para manejar tareas repetitivas de manera estructurada, como recorrer listas, realizar cálculos repetidos o ajustar valores hasta obtener un resultado deseado.

Busqueda Binaria

Algoritmo eficiente que encuentra un elemento en una lista ordenada dividiéndola repetidamente en dos mitades. En cada iteración, compara el valor buscado con el elemento central, descartando la mitad donde no puede estar. Este método reduce el tiempo de búsqueda significativamente, pasando de una complejidad lineal a una logarítmica.

Optimización

Proceso de ajustar un algoritmo o solución para hacerla más eficiente, reduciendo el tiempo de ejecución, el consumo de memoria o cualquier recurso crítico. Esto se logra mediante técnicas como eliminar redundancias, usar estructuras de datos más adecuadas o reestructurar el flujo del programa para alcanzar el mejor desempeño posible.

Prueba de Escritorio

Técnica en la que un programador simula manualmente la ejecución de un algoritmo paso a paso, utilizando ejemplos concretos de entrada. Se anotan los valores de las variables, las decisiones tomadas y los resultados en cada etapa. Es una herramienta valiosa para identificar errores lógicos y comprender el comportamiento de un algoritmo antes de implementarlo.

Complejidad Computacional

Rama de la informática que mide el rendimiento de un algoritmo evaluando el tiempo y el espacio necesarios para ejecutarlo, dependiendo del tamaño de la entrada. Clasifica los algoritmos según su eficiencia y comportamiento a medida que crece el problema, ayudando a elegir las soluciones más prácticas para cada situación.
